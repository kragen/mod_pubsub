When the user tabs out of the customer number field a PubSub event is sent to the app server which looks up the customer record and returns results via a PubSub event. This happens without any reloading of the page. Information is both sent and received via the PubSub Form component.


This form needs to be submitted to an external CGI that can communicate with the database and return new information in a new page. This style of interaction is both cumbersome and quite foreign to users accustomed to the rich UI of desktop applications.
With the use of PubSub's PubSub Form component, this form has been PubSub enabled simply by changing the action. With the user tabs out of this field the customer number will be wrapped into a PubSub event and sent to the server for processing.
Above we saw the PubSub Form component send information, and in this example the same component is allowing a normal HTML form to receive live information via PubSub.
