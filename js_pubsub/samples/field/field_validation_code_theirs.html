The old way

<html>
<body>

<form action="request.cgi" method="POST">
CUSTOMER NUMBER:<br />
<input type="text" name="number">

CUSTOMER NAME:<br />
<input type="text" name="name">
<br /><br />    

CUSTOMER ADDRESS:<br />
<input type="text" name="address">

<br /><br />

<input type="submit">
</form>

</body>
</html>
After the user fills out the form and submits it to request.cgi (a Perl, PHP, Python, JSP, ASP or Java application) a second page is returned with the results. Every time the user requests information from the server or needs to verify data, a new page will be loaded.


This form needs to be submitted to an external CGI which can communicate with the database and return new information in a new page. This style of interaction is both cumbersome and quite foreign to users accustomed to the rich UI of desktop applications.
With the use of the PubSub Form component, this form has been PubSub enabled simply by changing the action. With the user tabs out of this field the customer number will be wrapped into a PubSub event and sent to the server for processing.
Above we saw the PubSub Form component send information, and in this example the same component is allowing a normal HTML form to receive live information via PubSub.
