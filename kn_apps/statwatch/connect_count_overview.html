mod_pubsub

publish and subscribe messaging for the web.

Count connections

This counter shows the number of clients connected to the mod-pubsub.org server right now. It's the very simplest introduction to working with mod-pubsub.

Each mod-pubsub server keeps track of how many connections have been opened and closed, and whenever that number changes it POSTs an update to the /kn_statistics/liveconns topic. We use a small JavaScript library, kn_lib, to help us get that data as a JavaScript object via the kn_subscribe() function, which is then dropped into the IFRAME by means of the innerHTML method.

Why does the box say "Updates on change" when it first appears? Due to the design of our system, we can only get data that arrives at the server after our JavaScript library establishes a connection. There is no good way to ask for a dump of pre-existing data. Therefore, we can't give you an initial count as soon as the widget is initialized -- only when a new connection is made, or an old one breaks off.
