What Is This?

BlogChatter is a window into weblog activity right now, at this moment. Pings to Blogchatter are instantly displayed, and only persisted in memory for a half-hour at most. It is a real-time event stream of weblog updates, similar to the data provided by weblogs.com, without polling or a static data format.

The box to the right is an example application which uses the BlogChatter feed. It displays the incoming weblog updates the moment they are received.

Cool! Can I play?

Sure! We're looking for a few good bloggers to start pinging us when they post to their weblogs, so we can get some data moving through it. The interface is a URL that accepts a weblogs.com-compatible XML-RPC ping. If you're using Movable Type as your weblog tool, you already have the ability to send this ping. This ping is transformed into an event and published to a topic at mod-pubsub.org.

This is a beta web service, primarily for testing. The plan is to allow people to ping the server indefinitely and see what happens. In the best case, we'll start building apps that do interesting things with the real-time data and perhaps someday recreate it as a standalone service for stuff like real-time blogrolls.

How To Ping

To ping the interface from Movable Type, follow these steps:

  1. log in to MT and click Manage Blog for the blog you want to publish
  2. choose Blog Config
  3. go to Preferences
  4. scroll down to the section labeled "Publicity/Remote Interfaces/TrackBack."
  5. In the text box labeled "Others", add the following URL:

http://www.mod-pubsub.org/kn_apps/blogchatter/ping.php

Now save your changes. The next time you post to your weblog, the ping notification should happen automatically.

I don't use Movable Type, can I still ping from Blogger / Radio / Greymatter?

Sure, but we haven't had time to explore the capabilities of those tools, so if you have tips on how to configure them to ping BlogChatter, please share them.
